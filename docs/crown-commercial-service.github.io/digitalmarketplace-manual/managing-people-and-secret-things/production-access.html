Requirements for access to production environments¶

There are a few requirements that must be met before a new developer is allowed to access production accounts, environments and data.

They must have:

  • at least BPSS clearance

  • finished the standard onboarding process for new team members

  • read the guidelines for accessing production

  • paired with another developer in the use of the production environment (see Pairing or shadowing topics)

  • sign off from the team’s Tech Lead or Technical Architect

Pairing or shadowing topics¶

A new developer should discuss each of the following topics with a developer who has production access and pair with them on a task related to the topic if relevant.

  • How to declare an incident, and how to handle it

  • Use of the internal API

  • Use of PaaS

  • Use of AWS

  • Restrictions and process for connecting to the production database
